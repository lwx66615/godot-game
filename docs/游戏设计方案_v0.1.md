# 《诡异末世》游戏设计方案 v0.1

> 文档创建日期：2026/04/22
> 文档状态：初版讨论中
> 当前阶段：新手村（学校）Demo设计

---

## 一、项目概述

### 1.1 核心定位

**游戏类型：** 2D像素RPG + 生存 + 恐怖氛围

**核心体验：**
- 在诡异降临的末世世界中生存
- 探索末世降临的秘密
- 通过理解诡异规律来应对威胁

**参考作品：**
| 作品 | 借鉴内容 |
|------|----------|
| 《星露谷物语》 | 资源收集、生存发展、日夜循环 |
| 《Darkwood》 | 白天探索/夜晚防守、氛围叙事、无力感 |
| 《守墓人》 | 黑色幽默、经营管理、代价系统 |
| 《神秘复苏》 | 诡异设定、规律机制、关押概念 |

### 1.2 核心设计理念

| 理念 | 说明 |
|------|------|
| 规律对抗 | 诡异无法被消灭，只能通过理解规律来规避或关押 |
| 渐进认知 | 玩家与角色同步认知诡异，从陌生到理解 |
| 代价与选择 | NPC命运、资源分配、探索决策都有代价 |
| 氛围优先 | 恐怖感来自"未知"和"无力"，而非惊吓 |

## 二、世界观设定

### 2.1 世界背景

- 世界经历了"诡异降临"，人类文明濒临崩溃
- 诡异是无法被杀死、无法被理解的存在
- 诡异遵循特定的"规律"行动
- 人类只能通过理解规律来规避、关押或驾驭诡异

### 2.2 诡异的特性

| 特性 | 说明 |
|------|------|
| 不可消灭 | 诡异无法被杀死，只能关押或规避 |
| 规律驱动 | 每种诡异有独特的触发条件和行为规律 |
| 污染性 | 接触诡异会被污染，污染积累导致死亡 |
| 条件触发 | 诡异的出现和攻击需要特定条件（如看到、听到、开门等） |

### 2.3 游戏阶段规划

| 游戏阶段 | 世界状态 | 资源类型 | 诡异存在感 | 开发优先级 |
|----------|----------|----------|------------|------------|
| 初期（新手村） | 诡异刚降临，旧世界残留尚存 | 搜刮为主 | 弱→中 | Demo核心 |
| 中期 | 诡异扩散，旧世界资源枯竭 | 诡异产物开始出现 | 中→强 | 后续版本 |
| 后期 | 诡异全面降临 | 诡异产物为主 | 强 | 后续版本 |

## 三、新手村设计（学校）

### 3.1 整体框架

| 元素 | 设计 |
|------|------|
| 地点 | 大昌一中（参考《神秘复苏》） |
| 时长 | 7天 |
| 角色身份 | 失忆学生，诡异降临的幸存者 |
| NPC数量 | 6人 |
| 核心目标 | 关押学校诡异 → 解锁后续地图 |

### 3.2 白天-黑夜机制

| 时段 | 玩家行为 | 设计目的 |
|------|----------|----------|
| 白天 | 探索、搜刮、与NPC交流、收集线索 | 安全窗口期，建立世界观 |
| 夜晚 | 躲避或冒险探索 | 诡异登场，核心体验 |

**夜晚外出动机：**
- 夜晚专属资源（影响NPC命运的道具/信息）
- 夜晚专属探索区域
- 任务时限压力

**夜晚探索设计（混合方案）：**

| 夜晚类型 | 玩家行为 | 设计目的 |
|----------|----------|----------|
| 第1-2晚 | 可选择探索宿舍楼一层（诡异在远处活动） | 熟悉夜晚机制，风险可控 |
| 第3晚 | NPC C的"观察实验"任务 | 学习诡异规律，初次理解诡异行为模式 |
| 第4-5晚 | 寻找影响NPC命运的道具/线索 | 高风险高收益 |
| 第6晚 | 准备关押陷阱，检查铁盒位置 | 任务驱动 |
| 第7晚 | 关押战 | 核心体验 |

**夜晚探索动机设计（关键线索分层）：**

| 内容 | 白天可获得 | 夜晚可获得 | 设计目的 |
|------|------------|------------|----------|
| 基础物资（食物/水） | ✅ | ✅（但风险大） | 白天优先 |
| NPC存活条件线索 | ❌ | ✅（特定区域） | 强制夜晚探索 |
| 诡异规律验证 | 观察（远距离） | 亲历（近距离） | 夜晚获得更准确信息 |
| 部分日记/研究笔记 | ❌（被锁住） | ✅（诡异活动打开某些门） | 利用诡异规律 |

**具体例子：**

| 线索 | 位置 | 白天状态 | 夜晚状态 | 获取方式 |
|------|------|----------|----------|----------|
| NPC C存活条件（完整研究笔记） | 教学楼三层图书馆 | 门被锁住 | 诡异活动使门"自动开关" | 趁诡异在远处时进入 |
| NPC E说服证据（成功关押证据） | 行政楼档案室 | 门被锁住 | 诡异活动范围覆盖行政楼时门被打开 | 夜晚探索行政楼 |
| 诡异规律验证 | 教学楼各层 | 远距离观察 | 近距离亲历 | 第3晚NPC C的观察实验 |

**设计目的：** 玩家"想要"夜晚探索，而非"被迫"；需要权衡冒险获得线索 vs 安全但NPC可能死亡；每次夜晚探索都是对诡异规律的理解应用。

### 3.3 诡异活动范围信息传递

**采用"环境暗示 + 规律推断"方式：**

| 信息渠道 | 具体设计 |
|----------|----------|
| 前一晚观察 | 玩家前一晚观察诡异，记录其活动区域 |
| NPC间接提示 | NPC说"昨晚我听到三层有声音"、"今天食堂的门一直在自己开关" |
| 环境变化 | 白天某些区域有诡异残留痕迹（门自动开关、地面划痕） |
| 规律推断 | 玩家已知规律，推断诡异可能的路径 |

## 四、第一个诡异设计

### 4.1 基本信息

| 属性 | 设计 |
|------|------|
| 名称 | 门影诡（暂定） |
| 来源 | 敲门诡 + 屠刀诡的结合 |
| 出现条件 | 夜晚实体化，白天只能影响环境 |

### 4.2 外观设计

- 人形轮廓，模糊不清，像"影子里走出来的人"
- 手中拖着一把生锈的刀，刀刃在地上划出痕迹
- 没有脸，只有一张"嘴"的轮廓

### 4.3 规律设计

| 条件 | 行为 |
|------|------|
| 白天 | 无法实体化，但会让某些门"自动开关"，制造噪音 |
| 夜晚 | 实体化，开始巡逻 |
| 发现玩家 | 追踪，但速度不快 |
| 玩家进入房间关门 | 开始"敲门"，敲门声越来越急促 |
| 玩家开门 | 直接死亡 |
| 玩家不开门 | 约30秒后破门而入 |
| 玩家持续移动 | 追踪速度加快 |
| 玩家静止不动 | 追踪速度变慢，可能丢失目标 |

**核心规律总结：**
1. 被"门"吸引——有门的地方它就会去
2. 敲门是"攻击前摇"——给玩家反应时间
3. 静止可以降低它的感知——给玩家躲避空间
4. 刀的拖地声是预警——玩家可以听声辨位

### 4.4 诡异活动范围演变

| 天数 | 活动范围 |
|------|----------|
| 第1-2晚 | 教学楼西侧（宿舍楼方向） |
| 第3-4晚 | 教学楼中部→东侧 |
| 第5-7晚 | 教学楼全部区域 |

## 五、躲避系统设计

### 5.1 核心原则

- **没有传统战斗**——诡异无法被消灭，只能躲避或关押
- **规律驱动**——躲避基于对诡异规律的理解，而非反应速度
- **信息不对称**——玩家更多是"听"到危险，而非"看"到

### 5.2 玩家感知诡异的方式

| 感知方式 | 说明 | 设计目的 |
|----------|------|----------|
| **听觉（核心）** | 拖刀声（距离越近越清晰）、敲门声（预警） | 核心预警机制，让玩家"听声辨位" |
| **视觉（辅助）** | 看到诡异的模糊轮廓（但不要看得太清楚） | 保持"未知"的恐惧感 |
| **环境暗示** | 门自动开关、灯光闪烁、地面划痕 | 诡异在附近的间接提示 |
| **理智值影响** | 理智低时可能出现"假警报"（看到不存在的诡异） | 增加心理压力 |

### 5.3 诡异感知玩家的方式

| 条件 | 诡异感知变化 |
|------|--------------|
| 玩家移动 | 感知增强，追踪速度加快 |
| 玩家静止 | 感知降低，可能丢失目标 |
| 玩家开门/关门 | 强烈吸引，诡异会前往 |
| 玩家奔跑 | 声音吸引诡异注意 |

**感知范围层级：**

| 感知层级 | 范围 | 诡异行为 |
|----------|------|----------|
| 无感知 | 远距离 | 正常巡逻 |
| 隐约感知 | 中距离 | 停下，朝玩家方向观察 |
| 确认感知 | 近距离 | 开始追踪 |
| 锁定追踪 | 视线内 | 快速接近 |

### 5.4 玩家躲避手段

| 躲避手段 | 操作 | 效果 | 风险 |
|----------|------|------|------|
| **静止不动** | 停止移动 | 降低诡异感知，可能丢失目标 | 诡异可能还是靠近 |
| **躲进房间** | 进入房间，关门 | 诡异会敲门，给玩家30秒反应时间 | 必须选择开门或等待破门 |
| **躲藏点躲藏** | 躲在柜子/桌子下 | 诡异进入房间可能找不到 | 诡异靠近时不能动，理智值持续下降 |

**躲藏点机制：**

| 设计点 | 说明 |
|--------|------|
| 躲藏点类型 | 柜子、桌子下、阴影角落 |
| 躲藏效果 | 诡异进入房间后，有一定概率找不到玩家 |
| 躲藏风险 | 诡异可能在躲藏点附近徘徊；玩家必须保持不动；理智值每秒下降 |
| 躲藏成功条件 | 诡异进入房间→徘徊→未发现→离开 |

### 5.5 逃脱道具系统

**设计理念：**

| 设计理念 | 说明 |
|----------|------|
| 道具来源 | 前人求生失败的产物，蕴含执念或诡异力量 |
| 道具性质 | 不是普通物品，而是"被诡异污染"或"承载执念"的存在 |
| 使用代价 | 使用道具本身也有风险（消耗理智、道具损坏） |
| 稀缺性 | 新手村只有2个，不可再生 |

**新手村逃脱道具：**

| 道具名称 | 来源故事 | 效果 | 使用代价 | 获取位置 |
|----------|----------|------|----------|----------|
| **沾满血迹的门把手** | 一个学生在诡异追杀时试图创造"门"来吸引诡异，但失败了。他的求生执念和血液让这个门把手变成了诡异产物 | 放置后会在墙上显现一个模糊的"门框轮廓"，诡异会被吸引过去（约15秒） | 使用后消耗理智值5%，门把手消失 | 教学楼二层某教室角落（前人躲藏处） |
| **断裂的护身符** | 一个幸存者尝试用某种方法保护自己，但护身符在诡异靠近时断裂。断裂的护身符残留了部分保护力量 | 使用后玩家对诡异的感知降低10秒（诡异"看不见"玩家） | 使用后护身符完全损坏，消耗理智值3% | 教学楼三层图书馆角落（前人研究处） |

**道具获取风险：**

| 道具 | 获取风险 |
|------|----------|
| 沾满血迹的门把手 | 位于教学楼二层，第4天后诡异活动范围覆盖此处，需要夜晚冒险获取 |
| 断裂的护身符 | 位于教学楼三层图书馆，图书馆白天被锁，需要夜晚诡异活动时进入 |

### 5.6 被发现后的逃脱可能性

| 情况 | 逃脱可能性 | 说明 |
|------|------------|------|
| 诡异刚发现玩家（中距离） | 高 | 静止或躲藏可能甩掉 |
| 诡异正在追踪（近距离） | 中 | 需要利用门、躲藏点或逃脱道具 |
| 诡异已经锁定（极近距离） | 低 | 必须使用逃脱道具或躲藏点 |

### 5.7 玩家死亡条件

| 死亡条件 | 说明 |
|----------|------|
| 被诡异直接攻击 | 触碰诡异或诡异攻击范围内 |
| 开门给敲门的诡异 | 规律触发死亡 |
| 理智值归零 | 精神崩溃，诡异感知+50%，几乎无法躲避 |
| 资源耗尽 | 饥饿/口渴/体力归零后持续扣减，最终死亡 |

### 5.8 死亡惩罚

- 死亡后保留"已获得的线索"
- 当天重来，资源重置
- 让死亡有意义，而非纯惩罚

## 六、关押机制设计

### 5.1 核心概念

利用诡异的规律，将其引入"无法逃脱的容器"

### 5.2 关键道具

| 道具 | 位置 | 作用 |
|------|------|------|
| 封印铁盒 | 食堂储藏室（第5天获得） | 关押诡异的容器 |
| 铁盒钥匙 | 教学楼三层校史室（第6天获得） | 锁上铁盒，完成关押 |

### 5.3 关押战流程（规律验证 + 压力执行）

**核心设计理念：** 关押战是玩家对诡异规律理解的"期末考试"，不是QTE，而是知识应用。

**操作流程：**

| 阶段 | 玩家操作 | 规律体现 | 失败条件 |
|------|----------|----------|----------|
| 1. 布置 | 选择房间、放置铁盒位置 | 铁盒会吸引诡异，需要选有门的房间 | 选错房间（无门/多门）导致无法关押 |
| 2. 激活 | 启动铁盒，发出吸引波动 | 诡异被吸引，开始靠近 | 激活太早，诡异还没进入最佳位置 |
| 3. 躲藏 | 选择角落、保持静止 | 静止降低诡异感知 | 移动被发现，诡异直接攻击 |
| 4. 等待 | 听声辨位（拖刀声），判断诡异位置 | 诡异进入铁盒需要时间 | 过早冲出，诡异未完全进入 |
| 5. 执行 | 冲出房间→关门→NPC D协助锁上 | 诡异被困后会挣扎，有时间窗口 | 动作太慢，诡异挣脱 |

**NPC D的牺牲机制：**

| 设计点 | 说明 |
|--------|------|
| D的角色 | D协助锁门是关押成功的必要条件（铁盒需要两人同时操作才能锁上） |
| 牺牲触发 | 锁门后，诡异挣扎导致房间震动，D无法逃脱，被诡异残留污染致死 |
| 情感设计 | D是"唯一信任玩家的人"，从第4天开始建立情感连接，牺牲是"成功的代价" |

**核心体验：** 玩家不是在"按按钮"，而是在"运用知识"——
- "我知道诡异会被门吸引" → 选择单门房间
- "我知道静止可以降低它的感知" → 躲在角落不动
- "我知道拖刀声是预警" → 听声判断时机

**这才是"理解规律对抗诡异"的核心玩法体现。**

### 5.4 关押战失败处理

**核心设计理念：**

| 理念 | 说明 |
|------|------|
| 失败有代价 | 不是简单的"重来"，而是每次失败都有后果 |
| 紧张感递增 | 每次重试，难度或代价都会增加 |
| 不至于绝望 | 玩家不会因为一次失败就彻底失败，但代价累积会让最终成功变得沉重 |

**详细失败条件：**

| 失败条件 | 触发原因 | 失败后果 |
|----------|----------|----------|
| 诡异未完全进入铁盒就关门 | 玩家时机判断错误 | 诡异挣脱，玩家被攻击（受伤） |
| 诡异进入铁盒后未及时锁门 | 玩家动作太慢 | 诡异挣脱，NPC D为保护玩家受伤 |
| 玩家被诡异发现并攻击 | 躲藏失败或移动被发现 | 玩家受伤，关押战中断 |
| 铁盒放置位置错误 | 选择无门或多门房间 | 诡异无法被吸引，关押战无法进行 |

**代价累积系统：**

| 重试次数 | 代价 | 说明 |
|----------|------|------|
| 第1次失败 | 玩家受伤（体力上限降低20%） | 可以重试，但有损伤 |
| 第2次失败 | NPC D受伤（D的行动变慢，最终锁门时间缩短） | D的状态恶化 |
| 第3次失败 | NPC D死亡 | D无法协助锁门，关押战难度大幅增加 |
| 第4次失败 | 玩家死亡 | 读取存档，代价保留 |

**代价的具体影响：**

| 代价 | 影响 |
|------|------|
| 玩家受伤 | 体力上限降低、移动速度变慢、无法奔跑 |
| NPC D受伤 | D的对话变短、锁门动作变慢、可能无法完成锁门 |
| NPC D死亡 | 关押战需要玩家独自完成，锁门需要更长时间（或需要找到替代方案） |

**NPC D死亡后的替代方案：**

| 替代方案 | 说明 |
|----------|------|
| 独自完成 | 玩家需要更快地完成锁门，但时间窗口更短 |
| 寻找其他NPC协助 | 如果C/E/F存活，可以请求协助（但效果不如D） |
| 使用逃脱道具 | 用"沾满血迹的门把手"创造临时门吸引诡异，争取时间 |

**其他NPC协助的效果：**

| NPC | 协助效果 |
|-----|----------|
| C（研究者） | 可以协助，但锁门速度比D慢，时间窗口更短 |
| E（怀疑者） | 可以协助，但可能犹豫，有失败风险 |
| F（希望者） | 可以协助，效果与C类似 |

**玩家死亡的处理机制：**

| 设计 | 说明 |
|------|------|
| 不死档 | 玩家不会因死亡而丢失整个存档，避免过度挫败 |
| 代价保留 | 关押战失败的代价（NPC D受伤/死亡）保留，让失败有意义 |
| 重新准备 | 玩家可以重新准备关押战，但需要在已有代价的条件下 |

**存档机制：**

| 时间点 | 自动存档 | 说明 |
|--------|----------|------|
| 第7天傍晚 | ✅ 自动存档 | 关押战准备阶段 |
| 关押战开始前 | ✅ 自动存档 | 铁盒放置后，诡异被吸引前 |
| 关押战过程中 | ❌ 不存档 | 避免玩家卡在失败状态 |

**玩家死亡后的处理：**

| 死亡情况 | 读取存档 | 代价处理 |
|----------|----------|----------|
| 第1-3次失败后死亡 | 读取"关押战开始前"存档 | NPC D受伤/死亡状态保留 |
| 第4次失败后死亡 | 读取"第7天傍晚"存档 | 所有代价保留，玩家需要重新准备 |

**失败结局：**

| 项目 | 内容 |
|------|------|
| 触发条件 | 关押战多次失败，无法继续（NPC D死亡+其他NPC全部死亡+玩家多次失败） |
| 结局画面 | 玩家独自站在校门口，诡异在身后徘徊 |
| 结局旁白 | "你没能关押诡异。学校永远成为了它的领地。你只能逃离，但你知道，它还在那里。" |
| 后续影响 | Demo结束，提示玩家"未能完成新手村目标" |

## 七、NPC系统设计

### 6.1 NPC角色定位

| NPC | 定位 | 提供信息 | 死亡方式 | 是否可存活 |
|-----|------|----------|----------|------------|
| A | 幸存者领袖 | 生存基础、资源管理、诡异基础认知 | 剧情杀（第4天） | ❌ 固定死亡 |
| B | 恐惧者 | 诡异目击描述、敲门规律线索 | 选择驱动（第4-5天） | ❌ 固定死亡（但玩家选择影响过程） |

| D | 牺牲者 | 关押陷阱操作方法、情感连接 | 第7天关押战协助锁门后牺牲 | ❌ 固定死亡（协助是关押成功的必要条件） |

**NPC死亡的意义设计：**

| NPC | 死亡意义 | 铺垫设计 |
|-----|----------|----------|
| A（领袖） | 安全感的崩塌 | 第1-3天A对玩家的照顾细节（分配物资、保护玩家、分享故事），让玩家感受到"被保护" |
| B（恐惧者） | 玩家无力感的体现 | 第1-3天B的恐惧状态，第4天崩溃想逃离，玩家选择"劝阻"或"放行"，但两种选择B都会死亡 |
| D（牺牲者） | 情感高潮 | 第4天开始增加D与玩家互动，让D成为"唯一信任玩家的人"，第7天协助锁门是关押成功的必要条件 |

### 6.2 NPC B详细设计（选择驱动死亡）

**时间线：**

| 时间 | 事件 | 玩家行为 | 结果 |
|------|------|----------|------|
| 第1-3天 | B处于恐惧状态，提供诡异目击描述 | 与B对话，获得敲门规律线索 | B的信息有价值但带有恐惧色彩 |
| 第4天傍晚 | B因恐惧崩溃，提出想逃离学校 | 选择：劝阻 或 放行 | 关键抉择点 |
| 第4天夜晚 | 若劝阻：B暂时留下，但精神崩溃；若放行：B离开后死亡 | — | 分支 |
| 第5天 | 若劝阻：B失踪；若放行：发现B尸体 | — | — |
| 第5晚 | 若劝阻：B精神崩溃，自己跑出去死亡 | — | 悲剧收尾 |

**两种选择的对比：**

| 选择 | 第4晚结果 | 第5天结果 | 玩家获得 |
|------|-----------|-----------|----------|
| 劝阻 | B留下，精神崩溃 | B失踪，第5晚死亡 | B多活一天，但玩家感受到"无力感" |
| 放行 | B离开 | 发现B尸体，获得B遗物（完整敲门规律描述） | B死亡，但玩家获得关键信息 |

**设计目的：** 两种选择都有代价——劝阻是情感上"想救"但"救不了"，放行是理性上"放他走"获得信息但失去NPC。体现"代价与选择"理念。

### 6.3 可存活NPC的条件

| NPC | 存活条件 | 存活后作用 |
|-----|----------|------------|
| C（研究者） | 玩家在第5-6晚找到"完整研究笔记" + 选择"让她留下协助" | 为后续地图提供诡异分析能力 |
| E（怀疑者） | 玩家在第4-5晚找到"成功关押证据" + 在对话中展示证据说服他 | 为后续地图提供物资/战斗支持 |
| F（希望者） | 玩家在第5-6晚找到"外界安全路线信息" + 选择"让她留下" | 为后续地图提供外界情报 |

### 6.4 NPC死亡时间线

| 天数 | 事件 | NPC状态 |
|------|------|---------|
| 第1天 | 玩家苏醒，遇到NPC A；A照顾玩家，分配物资 | 6人存活 |
| 第2天 | 与各NPC互动，收集信息；B处于恐惧状态 | 6人存活 |

| 第7天 | 关押战，NPC D协助锁门后牺牲（协助是关押成功的必要条件） | 1-2人存活（C和E/F） |

### 7.4 第4天：第一次死亡

### 7.7 第7天：关押战

|----------|------|------|----------|
| 核心1 | 饥饿值 | 影响体力恢复、行动效率 | 食物 |
| 核心2 | 口渴值 | 影响体力恢复、行动效率 | 水 |
| 核心3 | 体力值 | 影响行动次数、奔跑能力 | 休息、食物 |
| 隐藏 | 理智值 | 影响视野、听觉、诡异感知 | 安全环境、NPC陪伴 |

### 9.2 饥饿值

| 设计点 | 说明 |
|--------|------|
| 初始值 | 100% |
| 每日消耗 | 白天消耗10%，夜晚消耗5% |
| 消耗加速 | 探索、奔跑消耗加速 |
| 饥饿状态 | 50%以下：体力恢复减半；20%以下：行动效率降低；0%：开始扣除体力 |
| 恢复方式 | 食物（+20%~40%） |

**食物类型：**

| 食物 | 恢复量 | 获取地点 | 备注 |
|------|--------|----------|------|
| 饼干/面包 | +20% | 食堂、宿舍 | 常见 |
| 罐头 | +30% | 食堂储藏室 | 较少见 |
| 能量棒 | +40% | 自动售货机 | 稀有 |

### 9.3 口渴值

| 设计点 | 说明 |
|--------|------|
| 初始值 | 100% |
| 每日消耗 | 白天消耗15%，夜晚消耗10% |
| 消耗加速 | 探索、奔跑消耗加速 |
| 口渴状态 | 50%以下：体力恢复减半；20%以下：视野略微模糊；0%：开始扣除体力 |
| 恢复方式 | 水（+25%~50%） |

**水源类型：**

| 水源 | 恢复量 | 获取地点 | 备注 |
|------|--------|----------|------|
| 瓶装水 | +25% | 食堂、宿舍、自动售货机 | 常见 |
| 饮水机水 | +30% | 教学楼走廊 | 需要容器 |
| 净化水 | +50% | 食堂后厨 | 稀有 |

### 9.4 体力值

| 设计点 | 说明 |
|--------|------|
| 初始值 | 100% |
| 消耗方式 | 探索移动、奔跑、搬运物品 |
| 恢复方式 | 休息（+10%/小时）、食物（额外恢复） |
| 体力状态 | 50%以下：无法奔跑；20%以下：移动速度降低；0%：无法移动 |

**体力与行动系统：**

| 行动 | 体力消耗 |
|------|----------|
| 普通移动 | 低消耗 |
| 奔跑 | 高消耗 |
| 搜刮/翻找 | 中消耗 |
| 搬运物品 | 高消耗 |
| 躲避（静止） | 不消耗 |

### 9.5 理智值（隐藏资源）

| 设计点 | 说明 |
|--------|------|
| 初始值 | 100% |
| 消耗方式 | 接触诡异、看到尸体、夜晚独自探索、诡异入侵安全区 |
| 恢复方式 | 安全环境休息、与NPC对话、阅读日记 |
| 理智状态 | 80%以上：正常；50%-80%：视野偶尔模糊、听到幻听；20%-50%：视野经常模糊、幻听频繁；20%以下：严重幻觉 |

**理智值的作用：**

| 理智值 | 视野效果 | 听觉效果 | 诡异感知 |
|--------|----------|----------|----------|
| 80%-100% | 正常 | 正常 | 正常 |
| 50%-79% | 偶尔模糊 | 偶尔幻听 | 诡异感知+10% |
| 20%-49% | 经常模糊 | 频繁幻听 | 诡异感知+30% |
| 0%-19% | 严重模糊 | 持续幻听 | 诡异感知+50% |

### 9.6 资源压力与日程配合

| 天数 | 资源状况 | 设计目的 |
|------|----------|----------|
| 第1-2天 | 资源充足（NPC A分配） | 让玩家熟悉资源系统 |
| 第3-4天 | 资源开始紧张（NPC A死亡，分配停止） | 玩家需要主动搜刮 |
| 第5-6天 | 资源稀缺（大部分区域已搜刮） | 玩家需要冒险探索危险区域 |
| 第7天 | 资源极度稀缺（或已耗尽） | 增加关押战的紧张感 |

### 9.7 死亡惩罚

- 死亡后保留"已获得的线索"
- 当天重来，资源重置
- 让死亡有意义，而非纯惩罚

### 9.8 白天玩法设计（资源驱动）

**核心设计理念：**

| 理念 | 说明 |
|------|------|
| 体力是核心限制 | 白天探索消耗体力，体力耗尽必须休息或停止探索 |
| 资源产生压力 | 食物/水不足会影响行动效率，玩家需要优先寻找资源 |
| 决策自然产生 | 不是强制"选择做什么"，而是"资源决定你能做什么" |

**体力状态与行动效率：**

| 体力状态 | 行动效率 | 说明 |
|----------|----------|------|
| 80%-100% | 正常 | 无影响 |
| 50%-79% | 轻微下降 | 移动速度略慢 |
| 20%-49% | 明显下降 | 无法奔跑，搜刮效率降低 |
| 0%-19% | 严重下降 | 移动缓慢，几乎无法行动 |

**饥饿/口渴对体力的影响：**

| 资源状态 | 对体力的影响 |
|----------|--------------|
| 饥饿值/口渴值 50%以下 | 体力恢复减半 |
| 饥饿值/口渴值 20%以下 | 体力无法恢复，持续消耗 |
| 饥饿值/口渴值 0% | 体力持续下降，最终无法行动 |

**时间流逝：行动驱动**

| 行动类型 | 时间消耗（游戏内） | 说明 |
|----------|-------------------|------|
| 移动到相邻房间 | 5分钟 | 基础移动 |
| 搜刮/翻找 | 15分钟 | 查找物品 |
| 与NPC对话 | 10-30分钟 | 根据对话长度变化 |
| 休息 | 30分钟 | 恢复体力 |
| 使用物品 | 5分钟 | 吃东西、喝水等 |
| 阅读日记/笔记 | 10分钟 | 获取信息 |

**白天总时长：**

| 时段 | 游戏内时长 | 现实时间估算 |
|------|------------|--------------|
| 早晨 | 4小时（6:00-10:00） | 约15分钟 |
| 中午 | 4小时（10:00-14:00） | 约15分钟 |
| 傍晚 | 3小时（14:00-17:00） | 约10分钟 |
| 白天总计 | 11小时 | 约40分钟 |

**时间显示：**

| 设计 | 说明 |
|------|------|
| 不显示精确时间 | 通过环境光线变化暗示时段 |
| 时段提示 | 时段过渡时显示提示（如"太阳升起来了"、"太阳快下山了"） |
| 日记记录 | 日记中记录"今天是第X天" |

### 9.9 NPC资源分配机制

**设计理念：**

| 理念 | 说明 |
|------|------|
| NPC也需要资源 | NPC也会饥饿、口渴，需要玩家分配资源 |
| 资源有限 | 总资源不足以让所有人吃饱，玩家需要做出选择 |
| 影响NPC状态 | NPC资源不足会影响其状态和对话内容 |

**NPC资源状态：**

| NPC状态 | 说明 | 影响 |
|----------|------|------|
| 正常 | 资源充足 | 正常对话，提供完整信息 |
| 饥饿 | 资源不足 | 对话变短，可能抱怨 |
| 虚弱 | 资源严重不足 | 无法提供帮助，可能死亡 |

**资源分配与日程：**

| 天数 | 资源状况 | 分配压力 |
|------|----------|----------|
| 第1-3天 | NPC A管理资源，自动分配 | 无压力 |
| 第4天后 | NPC A死亡，玩家接管资源分配 | 开始有压力 |
| 第5-7天 | 资源稀缺，需要做出艰难选择 | 高压力 |

**NPC资源不足的后果：**

| NPC | 资源不足后果 |
|-----|--------------|
| C（研究者） | 无法提供完整研究信息，存活条件更难达成 |
| E（怀疑者） | 更加怀疑玩家，更容易离开 |
| F（希望者） | 更想离开学校寻找外界资源 |

**资源分配界面：**

| 设计 | 说明 |
|------|------|
| 分配时机 | 每天早晨，玩家可以选择给NPC分配资源 |
| 分配方式 | 选择NPC → 选择给多少食物/水 |
| 反馈 | NPC会感谢或抱怨 |

## 十一、UI与信息呈现系统

### 11.1 核心设计原则

| 原则 | 说明 |
|------|------|
| 氛围优先 | UI应该融入世界观，而非"游戏化" |
| 间接暗示 | 信息通过环境、声音、状态描述呈现，而非精确数值 |
| 渐进认知 | 玩家与角色同步认知，信息逐步解锁 |
| 日记承载 | 任务/线索通过日记系统记录，而非独立任务列表 |

### 11.2 资源状态显示

**设计方向：模糊状态描述 + 视觉暗示**

| 资源 | 显示方式 | 状态描述示例 | 视觉暗示 |
|------|----------|--------------|----------|
| 饥饿值 | 模糊状态描述 | "你感到饥饿"（50%以下）、"你饿得头晕"（20%以下） | 角色动画变化（走路变慢） |
| 口渴值 | 模糊状态描述 | "你口渴了"（50%以下）、"你渴得喉咙发干"（20%以下） | 视野边缘轻微模糊 |
| 体力值 | 模糊状态描述 | "你有些疲惫"（50%以下）、"你累得走不动了"（20%以下） | 移动速度变化、呼吸声变化 |
| 理智值 | 完全隐藏 | 不直接显示，玩家需要自己判断 | 视野模糊、幻听、假警报 |

**理智值的间接暗示：**

| 理智状态 | 视觉效果 | 听觉效果 | 其他暗示 |
|----------|----------|----------|----------|
| 80%-100% | 正常 | 正常 | 无 |
| 50%-79% | 视野偶尔模糊（几秒后恢复） | 偶尔听到不存在的敲门声 | 无 |
| 20%-49% | 视野经常模糊 | 频繁幻听（拖刀声、敲门声） | 假警报（看到不存在的诡异轮廓） |
| 0%-19% | 严重模糊，几乎看不清 | 持续幻听 | 频繁假警报，几乎无法辨别真实威胁 |

### 11.3 诡异信息呈现

**设计方向：间接暗示为主，不直接显示诡异位置**

| 信息类型 | 呈现方式 | 设计目的 |
|----------|----------|----------|
| 诡异位置 | 不直接显示，通过拖刀声、环境变化暗示 | 保持"未知"的恐惧感 |
| 诡异距离 | 声音大小（越近越清晰）、环境变化强度（门开关频率） | 让玩家"听声辨位" |
| 诡异状态 | 不显示，玩家需要通过观察判断 | 增加不确定性 |

**环境暗示类型：**

| 暗示类型 | 说明 | 强度变化 |
|----------|------|----------|
| 拖刀声 | 诡异移动时的金属拖地声 | 越近声音越清晰 |
| 敲门声 | 诡异发现玩家躲进房间后的敲门预警 | 敲门声越来越急促 |
| 门自动开关 | 诡异在附近时，某些门会自动开关 | 频率越高说明诡异越近 |
| 灯光闪烁 | 诡异靠近时灯光可能闪烁 | 闪烁越频繁说明诡异越近 |
| 地面划痕 | 诡异经过的地方留下的刀痕 | 刚出现的划痕说明诡异刚经过 |

### 11.4 地图系统

**设计方向：手绘风格 + 探索解锁**

| 设计点 | 说明 |
|--------|------|
| 初始状态 | 只有玩家当前位置周围可见 |
| 探索解锁 | 玩家走过的地方才会显示详细内容 |
| 地图风格 | 手绘风格，像玩家自己画的草图 |
| NPC位置 | 不直接显示，需要与NPC对话后在日记中记录 |
| 诡异位置 | 不显示 |
| 未探索区域 | 显示轮廓，但无房间名称 |

**地图显示层级：**

| 显示层级 | 说明 | 解锁条件 |
|----------|------|----------|
| 未解锁 | 显示区域轮廓，但无房间详情 | 未进入该区域 |
| 轮廓解锁 | 显示区域轮廓和大致结构，但无房间名称 | 进入该区域但未详细探索 |
| 详情解锁 | 显示所有房间名称、资源点 | 详细探索该区域 |

**地图界面设计：**

| 设计点 | 说明 |
|--------|------|
| 打开方式 | 按键打开，或通过日记系统查看 |
| 界面风格 | 纸张质感，手绘线条 |
| 当前位置 | 用小标记显示玩家当前位置 |
| 安全点标记 | 已解锁的安全点可以用特殊标记显示 |

### 11.5 时间信息呈现

**设计方向：环境暗示为主，不显示精确时钟**

| 信息 | 呈现方式 |
|------|----------|
| 白天/夜晚 | 环境光线变化、天空颜色变化 |
| 时段过渡 | 屏幕边缘轻微变化提示（早晨→中午→傍晚→夜晚），无精确时间显示 |
| 当前天数 | 在日记界面显示，安全点可能有日历 |

**时段视觉变化：**

| 时段 | 光线 | 天空 | 环境氛围 |
|------|------|------|----------|
| 早晨 | 柔和光线，逐渐变亮 | 晨光，淡蓝色 | 安静，有鸟鸣声 |
| 中午 | 明亮光线 | 晴朗，蓝天 | 正常 |
| 傍晚 | 光线变暗，暖色调 | 黄昏，橙红色 | 紧张感开始增加 |
| 夜晚 | 暗淡光线，月光 | 黑暗，星空 | 恐怖氛围，诡异活动 |

### 11.6 日记系统（核心UI）

**设计理念：**

日记系统是UI的核心，承载事件记录、NPC对话、线索整理等功能，而非独立的任务列表。

| 功能 | 说明 |
|------|------|
| 事件记录 | 自动记录重要事件（如"第4天，NPC A死亡"） |
| NPC对话摘要 | 与NPC对话后，关键信息被记录 |
| 线索整理 | 玩家收集的线索可以在日记中查看 |
| 地图查看 | 日记中可以查看已解锁的地图 |

**日记界面设计：**

| 设计点 | 说明 |
|--------|------|
| 界面风格 | 手写风格，纸张质感，像玩家自己写的日记 |
| 记录方式 | 不是"任务列表"，而是"今天发生了什么"、"某人告诉我什么" |
| 时间标记 | 按天数分页，玩家可以翻阅之前的内容 |
| 打开方式 | 按键打开，暂停游戏 |

**日记内容示例：**

```
第1天

今天我在教室醒来，记忆模糊。
遇到了一个叫A的人，他告诉我世界发生了"诡异降临"。
他说夜晚不要外出，有危险。
学校里还有其他幸存者，我应该去认识他们。

[地图] - 教学楼一层（已解锁）
```

```
第3天

NPC C告诉我诡异可能可以被关押。
她说图书馆有研究笔记，但图书馆被锁住了。
她提议今晚做一个"观察实验"，观察诡异的行为。

我决定参与实验。

[线索] - 封印铁盒位置：食堂储藏室
[线索] - 诡异规律：静止可以降低它的感知
```

**日记系统的用户体验考量：**

| 设计点 | 说明 |
|--------|------|
| 中断后继续 | 玩家中断几天后回来，可以通过日记回顾之前的内容 |
| 不强制查看 | 日记是可选的，玩家可以选择不看 |
| 自动记录 | 重要事件和对话自动记录，玩家不需要手动记录 |
| 可添加备注 | 玩家可以自己添加备注（可选功能） |

### 11.7 物品界面

**设计方向：简洁、沉浸**

| 设计点 | 说明 |
|--------|------|
| 物品栏 | 有限容量，网格显示 |
| 物品描述 | 点击物品后显示描述，描述风格像"玩家观察物品" |

```
[使用后会消耗理智值]
```

## 十三、音效与氛围设计

### 13.1 核心氛围定位

| 维度 | 定位 | 说明 |
|------|------|------|
| 恐怖类型 | 心理恐怖 + 氛围恐怖 | 不是惊吓，而是"未知"和"无力"带来的恐惧 |
| 情感基调 | 压抑、孤独、不安 | 世界崩塌后的无力感，诡异存在的压迫感 |
| 参考风格 | 《Darkwood》的极简音效 + 《神秘复苏》的规律暗示 | 少而精，每个声音都有意义 |

### 13.2 音效设计原则

| 原则 | 说明 |
|------|------|
| 少即是多 | 音效数量有限，但每个都有明确意义 |
| 环境优先 | 大部分时间是环境音，让玩家沉浸 |
| 信息承载 | 音效不只是氛围，还承载游戏信息（如诡异位置） |
| 对比设计 | 白天安静平和，夜晚压抑紧张，形成对比 |

### 13.3 环境音效系统

**白天环境音：**

| 音效类型 | 说明 | 触发条件 | 情绪作用 |
|----------|------|----------|----------|
| 风声 | 室外的风声，轻微 | 持续播放，音量随位置变化 | 孤寂感 |
| 鸟鸣 | 偶尔的鸟叫声 | 随机触发 | 一丝生机，反衬末世 |
| 脚步声 | 玩家自己的脚步声 | 玩家移动时 | 基础反馈 |
| 门吱呀声 | 开关门的声音 | 玩家开关门时 | 基础反馈 |
| 寂静 | 几乎没有声音 | 室内空房间 | 孤独感 |

**夜晚环境音：**

| 音效类型 | 说明 | 触发条件 | 情绪作用 |
|----------|------|----------|----------|
| 风声 | 审外的风声，比白天更凄厉 | 持续播放 | 压抑感 |
| 寂静 | 极度安静，偶尔有电流声 | 室内 | 紧张感 |
| 远处拖刀声 | 诡异在远处活动时的金属拖地声 | 诡异在玩家视野外活动 | 核心预警，让玩家知道诡异存在 |
| 门自动开关声 | 诡异影响环境时门自动开关 | 诡异在附近 | 间接暗示诡异位置 |
| 灯光滋滋声 | 灯光闪烁时的电流声 | 诡异靠近时 | 紧张感 |

### 13.4 诡异音效系统（核心）

**诡异音效是玩家"听声辨位"的核心：**

| 音效类型 | 说明 | 距离变化 | 信息承载 |
|----------|------|----------|----------|
| **拖刀声** | 诡异移动时刀拖在地上的金属摩擦声 | 远：微弱模糊；近：清晰刺耳 | 诡异位置、移动方向 |
| **敲门声** | 诡异发现玩家躲进房间后的敲门 | 开始缓慢，越来越急促 | 攻击预警，给玩家反应时间 |
| **破门声** | 诡异破门而入 | 30秒敲门后触发 | 最后预警 |
| **徘徊声** | 诡异在房间内徘徊寻找玩家 | 诡异进入房间后 | 诡异在房间内的位置 |
| **静止声** | 诡异停下时的声音 | 诡异停止移动时 | 诡异可能在观察 |

**拖刀声的距离衰减设计：**

| 距离 | 音量 | 清晰度 | 附加效果 |
|------|------|----------|----------|
| 极远（3屏以上） | 几乎听不到 | 模糊 | 无 |
| 远（2-3屏） | 微弱 | 模糊 | 无 |
| 中（1-2屏） | 可听到 | 较清晰 | 玩家开始警觉 |
| 近（1屏内） | 清晰 | 清晰 | 紧张感增加 |
| 极近（同房间外） | 刺耳 | 非常清晰 | 高度紧张 |
| 同房间 | 极度刺耳 | 非常清晰 | 危机感 |

**敲门声的节奏设计：**

| 阶段 | 敲门节奏 | 持续时间 | 玩家状态 |
|------|----------|----------|----------|
| 第1阶段 | 缓慢，约2秒一次 | 0-10秒 | 可以观察、准备 |
| 第2阶段 | 加快，约1秒一次 | 10-20秒 | 需要做出决定 |
| 第3阶段 | 急促，约0.5秒一次 | 20-30秒 | 最后反应时间 |
| 破门 | 连续撞击声 | 30秒后 | 无法逃脱 |

### 13.5 玩家状态音效

| 音效类型 | 说明 | 触发条件 |
|----------|------|----------|
| 心跳声 | 玩家紧张时的心跳 | 诡异靠近时，音量随距离变化 |
| 呼吸声 | 玩家的呼吸声 | 正常时轻微，奔跑后急促，躲藏时压抑 |
| 脚步声 | 玩家的脚步声 | 移动时，奔跑时更响 |
| 状态提示音 | 资源状态变化时的提示 | 饥饿、口渴、体力低时 |

**心跳声设计：**

| 情况 | 心跳节奏 | 音量 |
|------|----------|------|
| 正常 | 无 | 无 |
| 诡异在中距离 | 缓慢 | 微弱 |
| 诡异在近距离 | 加快 | 可听到 |
| 诡异在同房间外 | 急促 | 清晰 |
| 躲藏时诡异在附近 | 极快 | 清晰，增加紧张感 |

### 13.6 理智值音效（幻听系统）

| 理智状态 | 音效表现 |
|----------|----------|
| 80%-100% | 无异常 |
| 50%-79% | 偶尔听到不存在的敲门声（幻听） |
| 20%-49% | 频繁幻听（拖刀声、敲门声），难以辨别真假 |
| 0%-19% | 持续幻听，几乎无法辨别真实声音 |

**幻听设计要点：**

- 幻听与真实声音相似，但略有不同（如更模糊、更短促）
- 玩家无法直接分辨真假，需要通过其他信息判断
- 增加心理压力，让玩家怀疑自己的感知

### 13.7 时段过渡音效

| 过渡 | 音效 | 情绪作用 |
|------|------|----------|
| 早晨→中午 | 鸟鸣渐弱 | 正常过渡 |
| 中午→傍晚 | 风声渐起 | 紧张感开始 |
| 傍晚→夜晚 | 风声变凄厉，远处传来拖刀声 | 恐怖氛围降临 |
| 夜晚→早晨 | 鸟鸣渐起，拖刀声消失 | 安全感回归 |

### 13.8 剧情与关押战音效

**NPC相关：**

| 音效类型 | 说明 | 触发条件 |
|----------|------|----------|
| NPC对话 | 纯文字，无语音 | 与NPC对话时 |
| NPC死亡 | 诡异的寂静或诡异的声音，非惨叫 | NPC死亡时 |

**关押战音效：**

| 阶段 | 音效 | 情绪作用 |
|------|------|----------|
| 激活铁盒 | 低沉的波动声 | 诡异开始被吸引 |
| 诡异靠近 | 拖刀声越来越近 | 紧张感 |
| 诡异进入铁盒 | 金属摩擦声 | 关键时刻 |
| 锁门 | 锁扣声、铁盒震动声 | 成功感 |
| 诡异挣扎 | 铁盒剧烈震动声、诡异低吼 | 最后危机 |
| 平静 | 震动停止，寂静 | 成功，释放感 |

### 13.9 音乐设计（BGM）

**设计原则：**

| 原则 | 说明 |
|------|------|
| 极简 | 音乐不是主角，氛围音效才是 |
| 环境化 | 音乐更像环境音，而非传统BGM |
| 动态 | 根据游戏状态变化，而非循环播放 |

**音乐类型：**

| 类型 | 说明 | 触发条件 | 风格参考 |
|------|------|----------|----------|
| 白天环境 | 极简的钢琴或弦乐，低音为主 | 白天，安全区域 | 《Darkwood》的安全区音乐 |
| 夜晚环境 | 低沉的 drone 音效，压抑感 | 夜晚，诡异活动 | 《Darkwood》的夜晚氛围 |
| 紧张状态 | 心跳节奏感的低频音效 | 诡异靠近时 | 动态增加紧张感 |
| 关押战 | 逐渐增强的节奏感音乐 | 关押战时 | 高潮体验 |
| 结局 | 悲伤但有一丝希望的旋律 | 游戏结束时 | 情感释放 |

**音乐动态变化：**

| 状态变化 | 音乐变化 |
|----------|----------|
| 白天→夜晚 | 音乐逐渐消失，转为环境音效 |
| 安全→诡异靠近 | 加入低频紧张音效 |
| 诡异远离 | 紧张音效逐渐消失 |
| 关押战开始 | 音乐逐渐增强 |
| 关押战成功 | 音乐达到高潮后逐渐平静 |

### 13.10 音效技术需求

| 需求 | 说明 |
|------|------|
| 3D音效 | 拖刀声需要根据诡异位置变化左右声道和距离衰减 |
| 动态混音 | 不同音效层次需要动态调整音量 |
| 幻听系统 | 理智值低时需要生成假音效 |
| 时段过渡 | 音效需要根据时段平滑过渡 |

## 十五、多结局设计

### 15.1 核心设计理念

| 理念 | 说明 |
|------|------|
| 无好坏之分 | 所有结局都是"灰色"的，只是方向不同 |
| 行为驱动 | 结局由玩家的行为累积决定，而非单一选择 |
| 反映风格 | 结局反映玩家的游戏风格和态度 |

### 15.2 玩家行为追踪

**需要追踪的行为类型：**

| 行为类型 | 具体行为 | 追踪方式 |
|----------|----------|----------|
| **探索行为** | 夜晚探索次数、探索危险区域、主动寻找线索 | 计数 |
| **理解行为** | 参与NPC C实验、收集诡异规律线索、阅读研究笔记 | 计数 |
| **情感行为** | 与NPC对话次数、关心NPC命运的选择、帮助NPC | 计数 |
| **生存行为** | 资源收集效率、躲避成功率、死亡次数 | 计数 |

**行为追踪转化为"倾向值"：**

| 倾向 | 计算方式 | 说明 |
|------|----------|------|
| **理解倾向** | 探索行为 + 理解行为 | 玩家是否主动理解诡异 |
| **情感倾向** | 情感行为 + 帮助NPC的选择 | 玩家是否关心他人 |
| **生存倾向** | 生存行为 + 自保选择 | 玩家是否专注于自身 |

### 15.3 主要结局设计

**结局A：理解者**

| 项目 | 内容 |
|------|------|
| 触发条件 | 理解倾向 ≥ 60%，且为三个倾向中最高 |
| 结局画面 | 玩家独自站在校门口，回头看了一眼学校，然后转身离开 |
| 结局旁白 | "你理解了诡异的规律，你知道如何在这个末世生存。但你也是孤独的——没有人能分享你的知识，没有人能理解你看到的真相。" |
| 存活NPC | 可能0-2人，但玩家选择独自前行 |
| 后续影响 | 为完整游戏提供"诡异分析"能力加成 |

**结局B：守护者**

| 项目 | 内容 |
|------|------|
| 触发条件 | 情感倾向 ≥ 60%，且为三个倾向中最高 |
| 结局画面 | 玩家与存活NPC一起站在校门口，NPC D的牺牲让所有人沉默 |
| 结局旁白 | "你失去了很多人，但你保护了一些人。在这个末世，有人陪伴是一种奢侈。你们一起走向未知的未来。" |
| 存活NPC | 1-2人，与玩家一起离开 |
| 后续影响 | 为完整游戏提供NPC同伴支持 |

**结局C：幸存者**

| 项目 | 内容 |
|------|------|
| 触发条件 | 生存倾向 ≥ 60%，且为三个倾向中最高；或三个倾向都低于60% |
| 结局画面 | 玩家独自站在校门口，表情麻木，没有回头 |
| 结局旁白 | "你活下来了。在这个末世，活着本身就是一种胜利。但你不知道自己失去了什么——也许是理解，也许是陪伴，也许是人性。" |
| 存活NPC | 0-2人，但玩家没有建立深刻连接 |
| 后续影响 | 为完整游戏提供"资源/生存"能力加成 |

### 15.4 隐藏结局：代价

| 项目 | 内容 |
|------|------|
| 触发条件 | 理智值在结局时低于20% |
| 结局画面 | 玩家站在校门口，视野模糊，听到幻听，不知道自己是否真的逃脱了 |
| 结局旁白 | "你活下来了，但你的一部分永远留在了那所学校。诡异在你脑海中低语，你不知道那是真实的，还是幻觉。" |
| 存活NPC | 任意 |
| 后续影响 | 为完整游戏提供"诡异感知"能力，但理智值上限降低 |

**设计说明：** 这个结局参考《神秘复苏》的"代价"概念——主角在接触诡异后，自身也变得"不正常"。

### 15.5 结局判定流程

```
1. 计算三个倾向值（理解、情感、生存）
2. 检查是否有倾向 ≥ 60%
   - 是 → 选择最高倾向对应的结局
   - 否 → 默认结局C（幸存者）
3. 检查理智值是否 < 20%
   - 是 → 覆盖为结局D（代价）
   - 否 → 保持原结局
```

### 15.6 结局与NPC存活的关系

**NPC存活影响结局细节，但不决定结局类型：**

| 结局 | NPC存活情况 | 结局细节变化 |
|------|-------------|--------------|
| 结局A（理解者） | 有NPC存活 | NPC目送玩家离开，玩家选择独自前行 |
| 结局A（理解者） | 无NPC存活 | 玩家独自离开，更加孤独 |
| 结局B（守护者） | 2人存活 | 玩家与2人一起离开，温暖但悲伤 |
| 结局B（守护者） | 1人存活 | 玩家与1人一起离开，更加悲伤 |
| 结局C（幸存者） | 有NPC存活 | NPC各自离开，玩家独自前行 |
| 结局C（幸存者） | 无NPC存活 | 玩家独自离开，完全孤独 |
| 结局D（代价） | 任意 | NPC存活不影响，玩家已被诡异影响 |

## 十六、存档系统设计

### 16.1 核心设计理念

| 理念 | 说明 |
|------|------|
| 增加紧张感 | 不允许随时存档，玩家需要规划探索 |
| 防止试错 | 不允许多存档，防止玩家反复尝试不同选择 |
| 保护进度 | 自动存档防止玩家丢失太多进度 |
| 融入氛围 | 存档界面融入日记系统，不破坏沉浸感 |

### 16.2 存档时机

| 存档类型 | 说明 | 设计目的 |
|----------|------|----------|
| **安全点存档** | 只能在安全点（办公室、接待大厅）存档 | 增加紧张感，玩家需要规划探索 |
| **时段自动存档** | 每个时段过渡时自动存档（早晨、中午、傍晚、夜晚） | 防止玩家丢失太多进度，一天4次 |
| **关键事件自动存档** | 关键事件后自动存档（NPC死亡、关押战等） | 确保重要节点有存档 |

### 16.3 存档数量

| 设计 | 说明 |
|------|------|
| 主存档 | 1个，玩家当前进度 |
| 自动备份 | 3个，自动存档的备份，玩家可选择回退 |
| 手动存档 | 不允许，防止玩家"试错"式游戏 |

**设计说明：** 不允许多存档是为了保护"代价与选择"的设计理念。如果玩家可以随时存档、读档、尝试不同选择，会削弱选择的重量感。自动备份用于防止意外（游戏崩溃、误操作）导致进度丢失。

### 16.4 存档内容

| 数据类型 | 具体内容 |
|----------|----------|
| 玩家状态 | 位置、资源值（饥饿/口渴/体力/理智）、物品栏 |
| 游戏进度 | 当前天数、当前时段、已完成的剧情事件 |
| NPC状态 | 各NPC存活状态、好感度、对话记录 |
| 地图数据 | 已解锁区域、已探索房间 |
| 物品数据 | 玩家物品栏、已使用的一次性道具 |
| 行为追踪 | 理解倾向、情感倾向、生存倾向（用于结局判定） |
| 日记数据 | 已记录的事件、对话摘要、线索 |

### 16.5 死亡与存档

| 设计 | 说明 |
|------|------|
| 死亡后 | 读取最近的自动存档（时段存档或事件存档） |
| 保留线索 | 死亡后保留"已获得的线索"（如日记中记录的信息） |
| 资源重置 | 死亡后当天的资源状态重置 |

**设计目的：** 让死亡有意义，而非纯惩罚。玩家虽然需要重新探索，但已获得的知识（线索、规律理解）保留，减少挫败感。

### 16.6 存档界面设计

| 设计 | 说明 |
|------|------|
| 存档界面 | 融入日记系统，存档时显示"写下今天的日记" |
| 存档提示 | 不显示"存档成功"，而是"你记录了今天发生的事" |
| 读档提示 | 不显示"读档成功"，而是"你翻开了之前的日记" |

### 18.2 状态详细定义

#### 18.2.1 巡逻状态（Patrol）

| 属性 | 设计 | 说明 |
|------|------|------|
| 移动速度 | 2.0 单位/秒（32像素/秒） | 缓慢，给玩家反应时间 |
| 路径选择 | 门优先 | 优先经过有门的区域 |
| 停顿频率 | 每10-15秒停顿3-5秒 | 模拟"观察"行为 |
| 拖刀声 | 持续播放，音量随距离衰减 | 核心预警 |

**巡逻路径逻辑：**

```
巡逻算法：
1. 获取当前活动范围内的所有"门"位置
2. 随机选择一个门作为目标
3. 沿最短路径移动到目标
4. 到达后，停顿3-5秒
5. 重复步骤1-4

特殊情况：
- 如果路径上有玩家，进入观察状态
- 如果听到开门/关门声，转向声源方向
```

**活动范围定义：**

| 天数 | 活动区域 | 巡逻节点数 |
|------|----------|------------|
| 第1-2晚 | 教学楼西侧（宿舍楼方向） | 4-6个门 |
| 第3-4晚 | 教学楼中部→东侧 | 8-10个门 |
| 第5-7晚 | 教学楼全部区域 | 12-15个门 |

#### 18.2.2 观察状态（Observe）

| 属性 | 设计 | 说明 |
|------|------|------|
| 触发条件 | 感知值达到"隐约感知"层级（21-50） | 见感知系统 |
| 行为 | 停止移动，朝玩家方向"看" | 模糊轮廓转向 |
| 持续时间 | 2-4秒 | 给玩家反应时间 |
| 拖刀声 | 停止 | 突然安静增加紧张感 |

**观察状态逻辑：**

```
观察算法：
1. 停止移动
2. 朝向感知到的玩家方向
3. 持续2-4秒
4. 如果感知值上升到"确认感知" → 进入追踪状态
5. 如果感知值下降到"无感知" → 返回巡逻状态
```

#### 18.2.3 追踪状态（Chase）

| 属性 | 设计 | 说明 |
|------|------|------|
| 基础速度 | 3.5 单位/秒（56像素/秒） | 比玩家行走快 |
| 追踪速度 | 4.0-5.5 单位/秒 | 玩家移动时加快 |
| 路径更新 | 每1秒更新一次 | 不是实时追踪 |
| 拖刀声 | 加快节奏 | 紧张感增加 |

**追踪速度计算：**

```
追踪速度 = 基础速度 + 移动加成

移动加成：
- 玩家静止：0
- 玩家行走：+0.5
- 玩家奔跑：+2.0

上限：5.5 单位/秒（88像素/秒）
```

**追踪逻辑：**

```
追踪算法：
1. 获取玩家当前位置
2. 计算最短路径
3. 沿路径移动
4. 每1秒重新计算路径
5. 如果玩家进入房间关门 → 进入敲门状态
6. 如果感知值归零 → 返回巡逻状态
```

#### 18.2.4 敲门状态（Knock）

| 属性 | 设计 | 说明 |
|------|------|------|
| 触发条件 | 玩家进入房间并关门 | 规律触发 |
| 持续时间 | 30秒 | 给玩家决策时间 |
| 敲门节奏 | 逐渐加快 | 三阶段设计 |

**敲门三阶段：**

| 阶段 | 时间 | 节奏 | 敲门间隔 | 音效 |
|------|------|------|----------|------|
| 第一阶段 | 0-10秒 | 缓慢 | 2秒/次 | 低沉、缓慢 |
| 第二阶段 | 10-20秒 | 加快 | 1秒/次 | 中等、急促 |
| 第三阶段 | 20-30秒 | 急促 | 0.5秒/次 | 高频、刺耳 |

**敲门状态逻辑：**

```
敲门算法：
1. 移动到玩家所在的门前
2. 开始敲门（三阶段节奏）
3. 如果玩家开门 → 玩家死亡
4. 如果30秒后玩家未开门 → 进入破门状态
```

#### 18.2.5 破门状态（Breach）

| 属性 | 设计 | 说明 |
|------|------|------|
| 触发条件 | 敲门30秒后 | 最后预警 |
| 持续时间 | 3秒 | 破门动画 |
| 音效 | 剧烈撞击声 | 危机感 |

**破门逻辑：**

```
破门算法：
1. 播放破门动画/音效
2. 3秒后门被破坏
3. 进入搜索状态
```

#### 18.2.6 搜索状态（Search）

| 属性 | 设计 | 说明 |
|------|------|------|
| 触发条件 | 进入房间后 | 寻找玩家 |
| 移动速度 | 1.5 单位/秒（24像素/秒） | 缓慢搜索 |
| 基础搜索时间 | 20秒 | 固定基础 |
| 随机变化 | ±5秒 | 最终范围15-25秒 |

**搜索逻辑：**

```
搜索算法：
1. 进入房间
2. 在房间内随机移动，检查躲藏点
3. 如果发现玩家 → 追踪状态
4. 如果15-25秒未发现 → 返回巡逻状态

躲藏点检测：
- 玩家躲藏在躲藏点：检测概率 30%-80%（根据躲藏点类型和玩家状态）
- 玩家未躲藏：100%发现
```

**躲藏点检测概率：**

| 躲藏点类型 | 玩家状态 | 检测概率 |
|------------|----------|----------|
| 柜子 | 完全静止 | 30% |
| 柜子 | 微小移动 | 50% |
| 桌子下 | 完全静止 | 40% |
| 桌子下 | 微小移动 | 60% |
| 阴影角落 | 完全静止 | 50% |
| 阴影角落 | 微小移动 | 80% |

### 18.3 感知系统详细设计

#### 18.3.1 感知值计算

**感知值范围：0-100**

| 感知层级 | 感知值 | 行为 |
|----------|--------|------|
| 无感知 | 0-20 | 正常巡逻 |
| 隐约感知 | 21-50 | 观察状态 |
| 确认感知 | 51-80 | 追踪状态 |
| 锁定追踪 | 81-100 | 快速追踪 |

#### 18.3.2 感知值来源

| 来源 | 感知值变化 | 说明 |
|------|------------|------|
| 玩家在视野内 | +10/秒 | 视线检测 |
| 玩家行走 | +3/秒 | 声音传播 |
| 玩家奔跑 | +8/秒 | 声音传播 |
| 玩家开门/关门（听觉范围内） | +30（瞬间） | 强烈吸引 |
| 玩家开门/关门（听觉范围外，同楼层） | +15（瞬间） | 中等吸引 |
| 玩家开门/关门（不同楼层） | +5（瞬间） | 微弱吸引 |
| 玩家静止 | -5/秒 | 感知衰减 |
| 玩家躲藏 | -10/秒 | 快速衰减 |
| 距离衰减 | -1~-5/秒 | 距离越远衰减越快 |

**距离衰减计算：**

```
距离衰减 = 基础衰减 × 距离系数

距离系数：
- 同房间：0（无衰减）
- 相邻房间：0.5
- 2-3房间距离：1.0
- 4-5房间距离：2.0
- 5房间以上：3.0
```

#### 18.3.3 感知范围

| 感知类型 | 范围（单位） | 范围（像素） | 范围（房间数） |
|----------|--------------|--------------|----------------|
| 视野范围 | 8单位（前方120度锥形） | 128像素 | 约0.5个房间 |
| 听觉范围 | 15单位（全方向） | 240像素 | 约1个房间 |
| 开门/关门感知 | 30单位 | 480像素 | 约2个房间 |

**视野检测：**

```
视野检测算法：
1. 计算诡异朝向与玩家方向的夹角
2. 如果夹角 < 60度（120度锥形）
3. 射线检测：诡异位置 → 玩家位置
4. 如果射线未被墙壁阻挡
5. 且距离 < 8单位
6. 则玩家在视野内
```

### 18.8 程序实现要点

#### 18.8.1 状态机实现建议

```gdscript
enum State { PATROL, OBSERVE, CHASE, KNOCK, BREACH, SEARCH }
var current_state = State.PATROL

func _process(delta):
    match current_state:
        State.PATROL: patrol_behavior(delta)
        State.OBSERVE: observe_behavior(delta)
        State.CHASE: chase_behavior(delta)
        State.KNOCK: knock_behavior(delta)
        State.BREACH: breach_behavior(delta)
        State.SEARCH: search_behavior(delta)
    
    update_perception(delta)
    check_state_transition()
```

#### 18.8.2 感知系统实现建议

```gdscript
var perception_value = 0  # 0-100

func update_perception(delta):
    # 视野检测
    if player_in_sight():
        perception_value += 10 * delta
    
    # 声音检测
    if player.is_walking():
        perception_value += 3 * delta
    elif player.is_running():
        perception_value += 8 * delta
    
    # 静止衰减
    if player.is_still():
        perception_value -= 5 * delta
    
    # 距离衰减
    var distance = get_distance_to_player()
    perception_value -= get_distance_decay(distance) * delta
    
    # 限制范围
    perception_value = clamp(perception_value, 0, 100)
```

## 十九、窗户系统设计

### 19.1 窗户类型

| 类型 | 说明 | 可打开 | 可逃出 |
|------|------|--------|--------|
| 普通窗户 | 教室常见 | ✅ | ✅ |
| 封闭窗户 | 部分房间 | ❌ | ❌ |
| 铁栅栏窗户 | 一层部分房间 | ✅ | ❌ |
| 破碎窗户 | 已损坏 | ❌（已开） | ✅ |

### 19.2 窗户逃出机制

| 条件 | 结果 |
|------|------|
| 窗户可打开 + 可逃出 | 玩家可从窗户逃出 |
| 窗户可打开 + 不可逃出（铁栅栏） | 玩家无法逃出，但可以观察窗外 |
| 窗户不可打开 | 玩家无法逃出 |

**逃出后果：**

| 楼层 | 逃出后果 |
|------|----------|
| 一层 | 安全逃出，无伤害 |
| 二层 | 逃出成功，体力-20%，可能受伤（30%概率） |
| 三层 | 逃出成功，体力-40%，受伤（100%概率） |

**受伤效果：**

| 伤害程度 | 效果 | 持续时间 |
|----------|------|----------|
| 轻伤 | 移动速度-10% | 当天 |
| 重伤 | 移动速度-30%，无法奔跑 | 当天 |

### 19.3 窗户与诡异的关系

| 情况 | 诡异行为 |
|------|----------|

## 二十五、教程/引导系统设计

### 25.1 设计原则

| 原则 | 说明 |
|------|------|
| 渐进式引导 | 不一次性灌输所有信息，随游戏进程逐步解锁 |
| 沉浸式引导 | 引导融入剧情和NPC对话，而非独立的教程界面 |
| 实践学习 | 让玩家通过实际操作学习，而非文字说明 |
| 可跳过 | 老玩家或二周目可跳过引导 |

### 25.2 第1天引导流程

#### 25.2.1 苏醒阶段（教室）

**触发条件：** 游戏开始

| 步骤 | 内容 | 玩家操作 | 引导目的 |
|------|------|----------|----------|
| 1 | 玩家在教室苏醒，视野模糊 | 无（自动） | 建立失忆设定 |
| 2 | 视野逐渐清晰，看到教室环境 | 移动视角 | 熟悉视角控制 |
| 3 | 提示"按WASD移动" | 移动 | 学习移动 |
| 4 | 玩家自由探索教室，发现物品 | 自由探索 | 自主学习交互 |
| 5 | NPC A出现，主动接近玩家 | 对话 | 建立NPC关系 |

**引导文本示例：**

```
[苏醒]
你的头很痛，记忆模糊...
这是哪里？

[移动提示]
按 WASD 移动
```

#### 25.2.2 遇到NPC A阶段

**触发条件：** 玩家离开教室或一段时间后

| 步骤 | 内容 | 玩家操作 | 引导目的 |
|------|------|----------|----------|
| 1 | NPC A主动接近玩家 | 无（自动） | 建立NPC关系 |
| 2 | A与玩家对话，解释基本情况 | 对话 | 了解世界观、获得基础地图 |

## 二十七、白天夜晚节奏平衡设计

### 27.1 核心问题分析

**当前设计的问题：**

| 问题 | 说明 |
|------|------|
| 白天太安全 | 诡异不实体化，玩家没有直接威胁 |
| 夜晚太危险 | 诡异实体化，玩家可能不敢外出探索 |
| 节奏断层 | 白天"跑腿"，夜晚"躲藏"，缺乏连贯的紧张感 |

**设计目标：**

| 目标 | 说明 |
|------|------|
| 白天有压力 | 不是直接威胁，而是资源、时间、心理压力 |
| 夜晚有选择 | 不是纯粹躲避，而是有策略性的探索机会 |
| 傍晚有过渡 | 不是突然切换，而是渐进的紧张感累积 |

### 27.2 时长设计

| 时段 | 现实时间 | 游戏内时间 | 说明 |
|------|----------|------------|------|
| 早晨 | 10分钟 | 4小时（6:00-10:00） | NPC分配、计划当天行动 |
| 中午 | 15分钟 | 4小时（10:00-14:00） | 主要探索时间 |
| 傍晚 | 10分钟 | 3小时（14:00-17:00） | 过渡阶段，返回安全点 |
| 夜晚 | 25分钟 | 7小时（17:00-24:00） | 诡异活动，策略性探索 |
| **总计** | **60分钟/天** | **18小时** | 一天约1小时现实时间 |

### 27.3 白天威胁设计

#### 27.3.1 白天威胁类型

| 威胁类型 | 具体设计 | 压力来源 |
|----------|----------|----------|
| **资源压力** | 食物/水有限，需要搜刮 | 生存压力 |
| **时间压力** | 白天时长有限（约35分钟现实时间） | 行动规划压力 |
| **体力压力** | 探索消耗体力，体力耗尽必须休息 | 行动限制 |
| **心理压力** | 门自动开关、灯光闪烁、诡异声音暗示 | 氛围压力（不扣数值） |
| **剧情压力** | NPC事件（A死亡、B崩溃等） | 情感压力 |

#### 27.3.2 白天诡异影响

白天诡异不实体化，但会影响环境：

| 影响类型 | 触发条件 | 效果 | 心理作用 |
|----------|----------|------|----------|
| 门自动开关 | 随机（诡异在附近时频率更高） | 门自己开关，发出声音 | 提醒诡异存在 |
| 灯光闪烁 | 随机 | 灯光闪烁 | 增加不安感 |
| 地面划痕 | 诡异经过的地方 | 刀痕痕迹 | 暗示诡异路径 |
| 远处声音 | 随机 | 微弱的拖刀声 | 提醒诡异在附近 |

**注意：白天进入诡异活动区域不扣除理智值。白天的压力来自氛围暗示，而非数值惩罚。**

### 27.4 夜晚安全窗口设计

#### 27.4.1 夜晚安全区

| 区域 | 安全程度 | 说明 |
|------|----------|------|
| 安全区（办公室、接待大厅） | 高 | 诡异不会主动进入，除非被吸引 |
| 玩家所在房间（关门） | 中 | 诡异会敲门，给玩家反应时间 |
| 走廊/开放区域 | 低 | 诡异可能巡逻经过 |
| 诡异活动区域 | 极低 | 诡异主要活动区域 |

#### 27.4.2 夜晚初期窗口

| 天数 | 夜晚初期窗口 | 说明 |
|------|--------------|------|
| 第1-2晚 | 全图相对安全 | 诡异在远处活动，玩家可以自由探索 |
| 第3-4晚 | 区域安全窗口 | 诡异在某些区域活动，其他区域相对安全 |
| 第5-6晚 | 观察窗口 | 需要先观察诡异位置，找到它不在的区域 |
| 第7晚 | 无窗口 | 关押战，无安全探索时间 |

#### 27.4.3 夜晚风险来源

| 风险来源 | 说明 |
|----------|------|
| 诡异巡逻 | 诡异在固定活动范围内巡逻，可能经过玩家所在区域 |
| 玩家行动 | 玩家移动、开门等行为会吸引诡异 |
| 停留时间 | 停留时间越长，诡异经过的概率越高 |

**注意：诡异活动范围是固定的（根据天数），不会在夜晚内扩大。**

### 27.5 傍晚过渡设计

#### 27.5.1 傍晚阶段

傍晚是白天到夜晚的过渡，约10分钟现实时间：

| 时间 | 光线变化 | 声音变化 | 玩家状态 |
|------|----------|----------|----------|
| 傍晚开始 | 光线开始变暗，暖色调 | 风声渐起 | 可以继续探索 |
| 傍晚中期 | 光线明显变暗 | 远处传来微弱的拖刀声 | 建议返回安全点 |
| 傍晚末期 | 接近夜晚的暗度 | 拖刀声更清晰 | 应该返回安全点 |

#### 27.5.2 傍晚NPC提醒

| 天数 | 提醒来源 | 内容 |
|------|----------|------|
| 第1天 | NPC A | "天快黑了，快回来。" |
| 第2天 | NPC A | "今晚比昨晚更危险，小心。" |
| 第3天 | NPC A | "记住，晚上不要乱跑。" |
| 第4天 | NPC C（A已死） | "天快黑了，我们回安全点吧。" |
| 第5-6天 | 环境暗示 | 光线变暗、远处拖刀声（无NPC提醒） |
| 第7天 | NPC D | "今晚是唯一机会，准备吧。" |

## 二十九、可重玩性设计

### 29.1 设计原则

| 原则 | 说明 |
|------|------|
| 不为重玩而重玩 | 重玩应该有新的体验，而非重复劳动 |
| 尊重玩家时间 | 不强制多周目，一周目也能获得完整体验 |
| 有意义的差异 | 二周目的变化应该影响玩法，而非只是数值调整 |

### 29.2 重玩动机

| 动机 | 设计支持 |
|------|----------|
| 看到不同结局 | 4个结局（理解者、守护者、幸存者、代价） |
| 拯救不同NPC | 3个可存活NPC（C、E、F），有多种组合 |
| 尝试不同选择 | B的选择、E的选择、F的选择 |
| 发现隐藏内容 | 解锁线索和日记详情 |

### 29.3 NPC存活组合

**可存活NPC：C（研究者）、E（怀疑者）、F（希望者）**

| 存活组合 | Demo影响 | 完整版影响（预留） |
|----------|----------|-------------------|
| C存活 | 结局细节变化 | 诡异分析能力 |
| E存活 | 结局细节变化 | 物资/战斗支持 |
| F存活 | 结局细节变化 | 外界情报 |
| C+E存活 | 结局B更温暖 | 分析+支持 |
| C+F存活 | 结局B更温暖 | 分析+情报 |
| E+F存活 | 结局B更温暖 | 支持+情报 |
| C+E+F存活 | 结局B最温暖 | 全能力支持 |
| 无NPC存活 | 结局A/C更孤独 | 无 |

**Demo阶段：NPC存活只影响结局细节和对话内容。**

### 29.4 二周目设计

#### 29.4.1 新游戏流程

| 步骤 | 说明 |
|------|------|
| 1. 主菜单 | 显示"继续"和"新游戏"选项 |

| 4. 开始游戏 | 根据周目调整引导和难度 |

#### 29.4.2 二周目引导变化

### 29.7 不设计的内容

| 不设计 | 原因 |
|--------|------|
| 无限模式 | 会破坏游戏的叙事完整性 |
| 随机生成 | 会破坏精心设计的诡异规律 |
| 多存档 | 会削弱"代价与选择"的设计理念 |
| 商城/付费内容 | 独立游戏，不应有内购 |
| 多人模式 | 会破坏恐怖氛围 |

### 29.8 重玩价值总结

| 重玩次数 | 预期体验 |
|----------|----------|
| 1次 | 完整体验主线剧情，达成1个结局 |
| 2-3次 | 尝试不同选择，达成不同结局，解锁隐藏内容 |
| 4次+ | 完成成就，尝试不同NPC存活组合 |

**设计目标：大部分玩家玩2-4次即可体验全部内容，不会强制无限重玩。**
